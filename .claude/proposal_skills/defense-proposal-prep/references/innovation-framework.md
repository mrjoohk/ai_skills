# 혁신성·도전성 포지셔닝 프레임워크

방산 연구 제안서에서 심사위원을 설득하는 혁신성·도전성 서술 방법론.
"혁신성은 마지막에 붙이는 것이 아니라, 기술 선택 기준으로 앞에서 작동해야 한다"는 원칙을 따른다.

---

## 핵심 원칙

### 원칙 1: 혁신성은 대비(Contrast)에서 나온다
혁신성은 단독으로 존재하지 않는다. **"기존 기술 X가 한계 Y를 가지고 있기 때문에, 우리 기술 Z가 이를 해결한다"**는 구조가 있어야 설득력이 생긴다.

### 원칙 2: 도전성은 수치로 표현한다
"어렵다"는 주장 대신 "현재 세계 최고 수준이 A인데 우리는 B를 목표로 한다 (N배 향상)"처럼 정량화해야 한다.

### 원칙 3: 심사위원의 언어를 쓴다
혁신성 키워드는 RFP의 평가 기준 언어에서 가져와야 한다. RFP가 "AI 융합"을 강조하면 그 용어를 사용하고, "국내 최초"를 평가하면 그 프레임을 활용한다.

---

## 프레임워크 A: 혁신 서사 구조 (Narrative Arc)

혁신성 설명을 4막 구조로 구성한다.

```
1막 [현황]: "현재 [분야]에서는 [기존 기술]이 사용되고 있으나..."
2막 [한계]: "...이는 [조건]에서 [한계]의 문제가 있어, [결과적 문제]를 초래한다."
3막 [제안]: "본 연구는 [핵심 기술]을 통해 [해결 방법]으로 이를 극복한다."
4막 [효과]: "이를 통해 [정량적 성능 향상]을 달성하고, [운용상 효과]를 가능하게 한다."
```

**예시 (레이더 분야):**
```
1막: "현재 전술 레이더는 파형 고정 방식의 FMCW 파형을 사용하고 있으나,"
2막: "이는 전자전 위협 환경에서 재밍 취약성 문제가 있어, 생존성이 현저히 저하된다."
3막: "본 연구는 AI 기반 적응형 파형 제어 기술을 통해 실시간 위협 대응 파형을 생성한다."
4막: "이로써 재밍 환경에서도 탐지 성공률 85% 이상을 유지하며, 플랫폼 생존성을 획기적으로 향상시킨다."
```

## 프레임워크 B: 혁신 유형 분류

아래 6가지 유형 중 해당 연구의 혁신 유형을 선택하고, 각 유형에 맞는 근거를 제시한다.

### Type 1: 성능 도약 (Performance Breakthrough)
- 정의: 기존 최고 수준 대비 정량적으로 유의미한 성능 향상
- 근거 요소: 현재 최고 성능치 → 목표 성능치, 향상 비율
- 표현 방법: "세계 최고 수준(X) 대비 N% 향상"
- 주의: 비교 기준(출처, 연도, 조건)을 반드시 명시

### Type 2: 기술 융합 (Technology Convergence)
- 정의: 서로 다른 도메인의 기술을 결합하여 기존에 없던 솔루션 창출
- 근거 요소: 융합하는 기술 A + 기술 B, 단독으로는 불가능한 이유
- 표현 방법: "AI/ML과 [도메인 기술]의 융합으로 기존 방법으로는 불가능했던 [X]를 실현"
- 주의: 단순 조합이 아닌 "시너지 효과"를 설명해야 함

### Type 3: 문제 재정의 (Problem Reframing)
- 정의: 기존과 다른 방식으로 문제를 정의하여 더 효율적인 해결책 제시
- 근거 요소: 기존의 문제 정의, 새로운 문제 정의, 이로 인해 가능해지는 것
- 표현 방법: "기존 [방법]은 [A] 문제로 접근하였으나, 본 연구는 [B] 관점으로 재정의하여..."
- 주의: 재정의의 논리적 근거가 명확해야 함

### Type 4: 제약 극복 (Constraint Breaking)
- 정의: 지금까지 "불가능하다"고 여겨졌던 물리적·기술적 제약을 새로운 방법으로 극복
- 근거 요소: 기존의 제약 (왜 어렵다고 했는가), 극복 방법의 원리
- 표현 방법: "[물리적/수학적 한계]로 인해 기존에는 [X]가 불가능하였으나, [새 방법]으로 이를 가능하게 함"
- 주의: 제약의 존재와 극복 원리를 학술적으로 뒷받침해야 함

### Type 5: 운용 패러다임 전환 (Operational Paradigm Shift)
- 정의: 기술 자체보다 운용 개념/방식을 혁신적으로 바꾸는 것
- 근거 요소: 기존 운용 방식의 한계, 새로운 운용 개념이 가져오는 전술적 이점
- 표현 방법: "기존 [운용 방식]에서 [새 운용 개념]으로의 패러다임 전환을 가능하게 하는..."
- 주의: 기술 개발과 운용 개념이 유기적으로 연결되어야 함

### Type 6: 국내 최초/자립화 (Domestic First)
- 정의: 국내에서 최초로 개발하거나, 해외 의존에서 탈피하는 것
- 근거 요소: 현재 수입/해외 의존 현황, 국내 개발 성공 시 효과
- 표현 방법: "현재 전량 수입에 의존하는 [기술]을 국내 최초로 독자 개발하여..."
- 주의: "국내 최초" 주장은 사전 특허/선행연구 조사로 확인 필요

## 프레임워크 C: 도전성 매트릭스

도전성은 3가지 차원으로 평가하고 각각 근거를 제시한다.

| 도전성 차원 | 평가 질문 | 높음(●) 근거 | 중간(◐) 근거 | 낮음(○) 근거 |
|---|---|---|---|---|
| **기술적 난이도** | 세계적으로 미해결 문제인가? | 기존 논문에서 open problem으로 명시 | 해결되었지만 국내 적용 어려움 | 기술 존재하나 국산화 필요 |
| **일정 도전성** | 목표 TRL 달성이 기간 내 빠듯한가? | TRL 4→7을 3년 내 달성 (세계 선례 없음) | 어렵지만 유사 사례 존재 | 충분한 시간과 전례 있음 |
| **복잡도** | 다분야 융합·대형 시스템 통합인가? | 3개 이상 도메인 융합 + 실환경 검증 필요 | 2개 도메인 또는 검증 환경 제한적 | 단일 도메인, 표준 검증 가능 |

**도전성 표현 예시:**
```
기술적 난이도: "본 기술의 핵심인 [X]는 현재 IEEE [학회명] 분야에서
  'Grand Challenge'로 분류되며, 세계적으로 TRL 4 이상의 구현 사례가 없다."

일정 도전성: "TRL 3에서 TRL 7까지의 기간을 3년으로 단축하는 것은
  유사 기술인 [Y]의 개발 기간(8년)에 비해 60% 압축된 도전적 일정이다.
  이는 [구체적 방법]을 통해 달성 가능하다."
```

## 프레임워크 D: 혁신성·도전성 자가 검증 체크리스트

제안서 작성 전 아래 질문에 답하여 포지셔닝을 점검한다.

**혁신성 체크:**
- [ ] "기존 최고 수준"의 출처와 수치가 명확한가?
- [ ] "우리의 목표"가 기존 대비 정량적으로 개선되었는가?
- [ ] 혁신의 핵심 원리가 1~2문장으로 설명 가능한가?
- [ ] 유사 특허/선행연구 조사를 완료했는가?

**도전성 체크:**
- [ ] "왜 어려운가"에 대한 기술적/과학적 근거가 있는가?
- [ ] 도전적인 TRL 목표와 달성 근거가 논리적인가?
- [ ] "그럼에도 가능한 이유"가 설명되는가? (도전성 ≠ 불가능)
- [ ] 기존 시도 실패 사례 또는 한계 사례를 인용했는가?

**심사위원 관점 체크:**
- [ ] RFP 평가 항목의 언어를 사용하고 있는가?
- [ ] 혁신 서사가 4막 구조(현황→한계→제안→효과)를 따르는가?
- [ ] 발표 자료에서 "우리 기술의 핵심 메시지" 1개가 즉시 전달되는가?

|---|---|---|
| 레이더/전자전 | 적응형 파형, AI 기반 탐지, 인지 전자전 | 클러터 환경, 저피탐 표적, 실시간 처리 |
| 유도무기 | 복합유도, 자율 교전, 정밀타격 | 비협조 표적, GPS 재밍 환경, 말기 유도 |
| 통신·네트워크 | 전술 메쉬, 항재밍, AI 기반 스펙트럼 관리 | 전자기전 환경, 지연 요구 |
| AI·자율 | 설명가능 AI, 인간-기계 협업, 엣지 추론 | 실시간 처리, 소형화, 신뢰성 |
| 소재·구조 | 다기능 복합소재, 스텔스 소재 | 환경 내구성, 고온·고압 조건 |
| 기계·추진 | 고효율 추진, 소형·경량화 | 수명 요구, 진동·충격 환경 |
