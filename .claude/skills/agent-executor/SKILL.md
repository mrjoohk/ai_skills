# Agent Executor — Subagent Execution Skill

**The current conversation session acts as the orchestrator**.
It reads `agents/orchestration_plan.md` and
based on each agent role file (`.md`),
**actually spawns subagents through the Agent tool**,
collects results, and generates an integrated report.

> This skill takes the `agents/` files generated by `agent-orchestration` as input.
> If `agents/orchestration_plan.md` does not exist, run `/agent-orchestration` first.

---

## Execution Steps

### Step 1 — Pre-Flight Checks
Verify the following in order:

1. Check existence of `agents/orchestration_plan.md`
   - If missing: Advise "Please run `/agent-orchestration` first to generate role design files" and halt
2. Check existence of agent role files (`agents/agent-*.md`)
3. Identify agent dependencies → determine execution groups

### Step 2 — Spawn Subagents by Execution Group

According to the **execution group order** in `orchestration_plan.md`:

- **Same group = parallel spawn**: Call Agent tool multiple times in a single message
- **Different group = sequential execution**: Spawn next group after previous group completes

Content to pass when spawning each subagent:
```
Extract from role file (agents/agent-N-role.md):
  - task_prompt  → Task instructions to pass to subagent
  - inputs       → File list subagent must read
  - outputs      → File paths subagent will generate
  - skills_to_use → Skills for subagent to use
```

Subagent task prompt format:
```
[Content of task_prompt from role file]

You must save output files to the following paths:
<outputs list>

After completion, report results in this format:
STATUS: PASS | FAIL
OUTPUTS: <List of generated file paths>
SUMMARY: <One-line work result summary>
ISSUES: <Problems encountered (or none)>
```

### Step 3 — Collect and Validate Results

After each subagent completes:
1. Check reported STATUS (PASS / FAIL)
2. Verify output files actually exist
3. Review `acceptance_criteria` satisfaction
4. If FAIL occurs:
   - Record error details
   - Determine if retry is possible
   - Decide whether to proceed with next group (halt on blocking errors)

### Step 4 — Generate Integrated Report

After all agents complete, generate `reports/orchestration_report_<timestamp>.md`.
Use the **Report Template** below.

## Important Notes
- Spawn subagents using the **Agent tool** (no separate CLI command needed).
- Parallel spawn is performed by **calling Agent tool multiple times in a single response**.
- Subagents run in independent contexts, so **results are shared only via files**.
- The orchestrator (current session) does not intervene in subagent intermediate processes.
- Each subagent's task_prompt must be **self-contained** (no external context dependency).

See `references/reference.md` for file format details and `references/examples.md` for example runs.
