# GPU/HPC Guard - 한국어 가이드

## 스킬 개요

GPU 및 HPC 자원 위험을 감시하는 스킬입니다. VRAM/RAM 메모리 사용량을 추정하고, 연산 복잡도 핫스팟을 분석하며, OOM(Out of Memory) 위험 패턴을 탐지합니다. 청킹, 스트리밍, 오프로드 전략을 제안하여 메모리 제약 내에서 안정적인 처리를 보장합니다.

## 언제 사용하는가

다음과 같은 상황에서 사용합니다:
- 대형 텐서 연산 (NumPy, CuPy, PyTorch, TensorFlow, CUDA 커널)을 수행할 때
- 배치 처리, 행렬 연산, 이미지·신호·포인트클라우드 처리 시
- 멀티-GPU 환경으로 스케일링을 계획할 때
- 메모리 예산 초과 경고가 발생하는 계산 집약 모듈 분석 시

## 입력

다음 정보를 제공해야 합니다:
- **분석 대상:** 함수 또는 모듈 경로 + 심볼 참조 (예: `src/model/encoder.py::forward()`)
- **예상 데이터:** 배치 크기, 텐서 형태, 데이터 타입 (예: B=32, shape=(3, 512, 512), dtype=float32)
- **하드웨어 제약:** VRAM/GPU 수, PCIe 대역폭, RAM 한도

## 출력물

다음 산출물이 생성됩니다:

| 산출물 | 설명 |
|--------|------|
| **VRAM/RAM 추정 테이블** | 각 텐서별 바이트 수, dtype, shape, peak 메모리 위치 표시 |
| **복잡도 분석** | Big-O 표기, 지배 상수, Memory-bound/Compute-bound 구분 |
| **완화 계획** | 청크 크기, 스트리밍 단계, 커널 융합, 오프로드 지점 |
| **수락 기준** | 최대 메모리, UF당 최대 시간, 정확도 검증 체크리스트 |

## 슬래시 명령 예시

### 예시 1: VRAM 추정
```
배치 크기 B=32, 이미지 크기 (3, 512, 512), 중간 특징 채널 C=256,
dtype=float32 기준으로 모델 순전파의 VRAM 사용량을 추정하라.
24GB VRAM 한도 내에서 처리 가능한 최대 배치 크기를 제안하라.
```

### 예시 2: OOM 완화 계획
```
src/model/decoder.py::decode() 에서 OOM이 발생한다.
해당 함수 내 주요 텐서 할당을 분석하고,
peak VRAM을 16GB 이하로 유지하는 청킹/오프로드 계획을 제안하라.
```

### 예시 3: 멀티-GPU 분산 전략
```
현재 단일 GPU(40GB)에서 학습 중이며, 모델 크기가 35GB이다.
2-GPU 환경으로 확장할 때 데이터/모델/파이프라인 병렬화 중
어떤 전략이 적합한지 분석하라.
```

## 다른 스킬과의 연결

**함께 사용하는 스킬 흐름:**

1. **core-engineering** → **gpu-hpc-guard**
   - 설계 단계에서 UF/IF를 정의한 후, 각 함수의 메모리 리스크 감시

2. **gpu-hpc-guard** → **sim-physics-auditor**
   - 메모리 제약 내에서 수치 안정성 검증 필요 시

3. **gpu-hpc-guard** → **eval-runner**
   - 청킹/스트리밍 변형에 대해 성능 벤치마크 실행

## VRAM 추정 테이블 가이드

| 항목 | 설명 |
|------|------|
| **float32** | 4 bytes |
| **float16** | 2 bytes |
| **int8** | 1 byte |
| **complex64** | 8 bytes |

Peak 메모리 = Σ(동시 존재 텐서) + 프레임워크 오버헤드 (~10-20%)

## 완화 전략 플레이북

| 전략 | 조건 | 적용 방법 |
|-----|------|---------|
| **청킹** | 단일 텐서 > VRAM 한도 | 배치 또는 공간 차원 분할 |
| **스트리밍** | 파이프라인 순차 단계 | 계산/전송 오버랩 |
| **오프로드** | VRAM 부족, RAM 여유 | GPU→CPU 명시적 지점 |
| **커널 융합** | 중간 텐서 반복 생성 | 연속 연산 병합 |
| **혼합 정밀도** | float32 전체 불필요 | fp16/bf16 전환 가능 레이어 |
| **멀티-GPU** | 단일 GPU 용량 초과 | 데이터/모델/파이프라인 병렬화 |
