# Sim Physics Auditor - 한국어 가이드

## 스킬 개요

물리 시뮬레이션의 일관성과 수치 정확성을 검증하는 스킬입니다. 단위와 차원의 일관성, 좌표계 변환, 샘플링 이론 준수, 수치 안정성을 종합적으로 감시합니다. PASS/WARN/FAIL 체크리스트로 문제를 식별하고, 파라미터 스윕 실험을 통해 신뢰성을 검증합니다.

## 언제 사용하는가

다음과 같은 상황에서 사용합니다:
- 수치 시뮬레이션 결과를 신뢰하기 전에 (항상)
- 시뮬레이션 파라미터를 변경할 때 (주파수, 속도, 샘플링 레이트, 스텝 크기)
- 물리 방정식을 다른 언어·프레임워크로 포팅할 때 (Python → C++ → C#)
- 컨트롤 루프 또는 필터 설계를 변경할 때
- 신규 물리 도메인 모듈을 추가할 때 (신호처리, 동역학, 광학, 열, 제어)

## 입력

다음 정보를 제공해야 합니다:
- **물리 방정식:** 차원 분석, 수식, 가정(Assumptions)
- **파라미터 범위:** 주파수, 속도, 스텝 크기, 룩업 범위 등
- **기대 출력:** 해상도, 오차 범위, SNR 지표, 안정성 마진
- **도메인 힌트:** 신호처리 / 동역학 / 광학 / 열 / 제어 등

## 출력물

다음 산출물이 생성됩니다:

| 산출물 | 설명 |
|--------|------|
| **PASS/WARN/FAIL 체크리스트** | 단위, 좌표계, 나이퀴스트, 수치 안정성 항목 |
| **체크리스트 결과 테이블** | 각 항목별 상태, 의심 위치(경로::심볼), 비고 |
| **파라미터 스윕 테이블** | 파라미터 범위, 단계 수, 기대 출력 변화, 결과 경로 |
| **권장 검증 실험** | 재현 단계, 로그 수집, 중단점, 프로파일링 계획 |
| **수락 임계값** | 단위 오류: 0건, 수치 오차: <=eps, 안정성 마진 |

## 슬래시 명령 예시

### 예시 1: 단위·좌표계 감사
```
src/sensor/imu_processor.py 를 감사하라.
모든 변수의 단위 명시 여부와 deg/rad 혼용 여부를 확인하고,
PASS/WARN/FAIL 체크리스트를 출력하라.
문제가 있는 경우 최소 코드 변경(diff)을 제안하라.
```

### 예시 2: 나이퀴스트 조건 검증
```
샘플링 레이트 fs=1000Hz, 최대 신호 주파수 f_max=450Hz,
FIR 필터 차단 주파수 fc=400Hz 에 대해
나이퀴스트 조건과 대역폭-해상도 관계를 검증하라.
PASS/WARN/FAIL 체크리스트와 권장 검증 실험을 제안하라.
```

### 예시 3: 제어 루프 안정성 검증
```
src/control/pid_controller.py 의 50Hz 제어 루프를 감사하라.
수치 적분기 방법, 포화 처리, 안정성 조건을 확인하고
step response 및 saturation 테스트 실험을 제안하라.
```

### 예시 4: 언어 간 포팅 검증
```
Python과 C++ 구현 간 물리 방정식의 동등성을 검증하라:
Python: src/sim/dynamics.py::integrate()
C++:    src/sim/dynamics.cpp::integrate()
동일 입력에 대해 수치 오차가 eps=1e-6 이하임을 확인하는
검증 계획을 수립하라.
```

## 다른 스킬과의 연결

**함께 사용하는 스킬 흐름:**

1. **core-engineering** → **sim-physics-auditor**
   - 설계 단계에서 물리 방정식을 정의한 후 검증

2. **sim-physics-auditor** → **eval-runner**
   - 파라미터 스윕 실험 자동화 및 성능 벤치마크

3. **gpu-hpc-guard** + **sim-physics-auditor**
   - 메모리 제약 내에서 수치 안정성 동시 검증

## 지원 도메인

| 도메인 | 체크 항목 |
|--------|----------|
| **신호 처리** | 나이퀴스트 조건(fs >= 2×f_max), 대역폭·해상도 관계, 윈도잉·필터링 가정 |
| **동역학 / 제어** | 타임스텝 dt, 적분 방법(RK4/Euler), 포화 처리, 안정성 조건 |
| **광학 / 이미징** | 초점거리, 화각(FOV), 픽셀 크기, 좌표 원점(image vs sensor vs world) |
| **열 / 유체** | 온도 단위(K vs °C), 경계 조건, 초기 조건 |

## 공통 단위·좌표계 체크리스트

### 단위 (Units)
- [ ] 모든 변수에 단위가 명시되어 있다
- [ ] 단위 변환(deg↔rad, m↔km, Hz↔kHz 등)이 명시적으로 처리된다
- [ ] 입력·출력 단위가 함수 시그니처 또는 문서에 기재되어 있다

### 좌표계 (Coordinate Frames)
- [ ] 좌표계가 선언되어 있다 (예: ENU, NED, body, world, image)
- [ ] 좌표계 변환 지점이 명시적으로 표시된다
- [ ] 각도: deg/rad 혼용 없이 일관성이 유지된다

## 검증 출력 형식

### 체크리스트 결과 테이블
```
| 항목 | 상태 | 의심 위치 (경로::심볼) | 비고 |
|------|:----:|----------------------|------|
| 단위 일관성      | PASS/WARN/FAIL | ... | ... |
| 좌표계 선언      | PASS/WARN/FAIL | ... | ... |
| 나이퀴스트 조건  | PASS/WARN/FAIL | ... | ... |
| 수치 안정성      | PASS/WARN/FAIL | ... | ... |
```

### 수락 임계값
```
- 단위 오류:    0건 (FAIL 항목 없음)
- 수치 오차:    <= eps (해석해 또는 기준 케이스 대비)
- 안정성 마진:  명시된 조건 내 bounded 상태 유지
```
