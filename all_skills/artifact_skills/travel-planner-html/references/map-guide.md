## 초기 center & zoom 선택 기준

| 여행 범위           | zoom  | center 산출           |
|---------------------|-------|-----------------------|
| 도시 내 (≤5km)      | 13~14 | places lat/lng 평균   |
| 광역 근교 (≤30km)   | 10~11 | places lat/lng 평균   |
| 현(縣) 단위 (≤100km)| 8~9   | places lat/lng 평균   |
| 복수 도시           | 7~8   | places lat/lng 평균   |
| 국가 전체           | 5~6   | 국가 중심 좌표        |

center는 `places[]`의 lat 평균, lng 평균을 쓰면 안정적.

## 좌표 확인 방법

좌표는 **반드시** 아래 방법 중 하나로 확인한다. 기억에서 적으면 마커가 엉뚱한 곳에 찍힌다.

1. Google Maps에서 장소 검색 → URL의 `@37.xxxx,138.xxxx` 부분
2. 웹 검색: `"{장소명}" latitude longitude`
3. 확인 불가 장소는 `places[]`에서 제외 (타임라인 텍스트에만 남김)

## 주의사항

| 문제 | 원인 & 해결 |
|------|-------------|
| 마커에 흰 박스 겹침 | `L.divIcon({ className: '' })` — className 빈 문자열 필수 |
| 경로 선이 지그재그 | `places[]`가 방문 순서대로 정렬되지 않음 → day별 방문순 정렬 |
| 단일 장소 day에 오류 | `pts.length < 2` 이면 polyline 생성 건너뜀 처리 |
| filterDay에서 콘솔 오류 | `routes[day]`가 없는 day에 접근 → `if (!routes[day]) return` 가드 |
| 팝업이 마커 뒤에 숨음 | `iconAnchor:[17,34]`, `popupAnchor:[0,-36]` 값 유지 |
