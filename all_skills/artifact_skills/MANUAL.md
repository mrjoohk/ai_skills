# Artifact Skills — 스킬 연동 사용 매뉴얼

> **디렉토리 목적:** 독립 실행형 시각 아티팩트 생성. 사용자가 제공한 데이터(URL 목록, 여행 정보 등)를
> 인터랙티브 HTML 또는 복합 문서 형태로 즉시 산출한다.

---

## 포함 스킬

| 스킬 | 역할 | 주요 출력 |
|------|------|-----------|
| `tab-digest` | URL 목록 → 구조화 다이제스트 | HTML 대시보드 + Word(.docx) |
| `travel-planner-html` | 여행 계획 → 공유용 인터랙티브 플래너 | 단일 HTML 파일 (지도 + 일정 + 체크리스트) |

---

## 스킬별 단독 사용 흐름

### tab-digest

```
[입력] URL 목록 (붙여넣기 또는 파일)
    ↓
tab-digest 실행
    ↓
[출력] HTML 대시보드 + docx 요약 문서
```

**트리거 예시:**
- "이 링크들 정리해줘"
- "탭 너무 많아, 요약해줘"
- "저장해놓은 URL들 분석해줘"
- URL 목록을 그냥 붙여넣기

**핵심 출력:**
- 페이지별 TL;DR, 핵심 정보, 액션 아이템(설치 명령어, 다운로드 링크 등) 추출
- 카테고리별 필터링이 가능한 HTML 대시보드

---

### travel-planner-html

```
[입력] 여행 목적지, 기간, 일정 정보 (대화 또는 문서)
    ↓
travel-planner-html 실행
    ↓
[출력] {목적지}_travel.html (탭형 일정 + Leaflet 지도 + 체크리스트)
```

**트리거 예시:**
- "여행 계획 HTML로 만들어줘"
- "공유용 여행 플래너 파일 만들어줘"
- "일정 시각화해서 파일로 뽑아줘"

**핵심 출력:**
- 날짜별 탭 전환형 일정 타임라인
- 장소 핀이 찍힌 인터랙티브 지도
- 준비물/할일 체크리스트

---

## 두 스킬 연계 시나리오

두 스킬은 독립적으로 동작하나, 여행 계획 수립 시 아래 순서로 연계할 수 있다.

```
[여행 관련 링크 수집]
    ↓
1. tab-digest — 링크들 요약·핵심 정보 추출
   (숙소, 관광지, 맛집 정보 정리)
    ↓
2. travel-planner-html — 정리된 정보로 여행 플래너 HTML 생성
```

**연계 트리거 예시:**
- "이 여행 관련 링크들 먼저 정리하고, 그 내용으로 여행 계획 HTML 만들어줘"

---

## 스킬 선택 가이드

| 상황 | 사용할 스킬 |
|------|-------------|
| URL 여러 개를 받아 요약·정리하고 싶다 | `tab-digest` |
| 여행 일정을 시각화해서 공유하고 싶다 | `travel-planner-html` |
| 여행 관련 링크를 모아두고, 그것을 기반으로 여행 계획을 만들고 싶다 | `tab-digest` → `travel-planner-html` |

---

## 주의사항

- 두 스킬 모두 HTML을 핵심 출력 포맷으로 사용한다. 외부 CDN(Leaflet, Google Fonts)을 사용하므로 인터넷 연결이 필요하다.
- `tab-digest`는 WebFetch 접근이 불가한 URL(인증 필요, 차단된 도메인)의 경우 해당 페이지를 건너뛰고 처리 가능한 URL만 다이제스트한다.
- `travel-planner-html`의 지도 좌표는 Claude가 추론하므로, 정밀도가 중요한 경우 사용자가 직접 좌표를 확인하는 것을 권장한다.
