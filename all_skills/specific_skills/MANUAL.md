# Specific Skills — 스킬 연동 사용 매뉴얼

> **디렉토리 목적:** 특정 기술 도메인의 구현 결과물을 전문적으로 감사(audit)하는 스킬 묶음.
> 범용 파이프라인의 구성원이 아니라, **도메인 조건에 해당할 때 선택 투입**하는 감사 레이어다.

---

## 포함 스킬 목록

| 스킬 | 감사 도메인 | 주요 검사 항목 |
|------|------------|----------------|
| `gpu-hpc-guard` | GPU / HPC 연산 | VRAM 사용량 추정, 연산 복잡도 핫스팟, OOM 패턴, 청킹·스트리밍 전략 |
| `sim-physics-auditor` | 물리 시뮬레이션 | 단위·차원 검증, 좌표계 일관성, 수치 안정성, 샘플링 이론 준수 |
| `rag-data-quality` | RAG 코퍼스 / 임베딩 | 메타데이터 완결성, 청킹 규칙, 중복 탐지, 도메인 커버리지, Recall@k |

---

## 스킬별 독립 사용 가이드

### gpu-hpc-guard

**트리거 예시:**
- "GPU 메모리 괜찮아?", "OOM 날 것 같아"
- "HPC 연산 최적화해줘", "멀티 GPU 스케일링 검토"

**입력:**
```
분석 대상 함수/모듈 (경로 + 심볼)
기대 데이터 차원 (배치 크기, 텐서 shape, 데이터 타입)
하드웨어 제약 (VRAM, GPU 수, PCIe 대역폭, RAM 한도)
```

**출력:**
- VRAM/RAM 사용량 추정
- 연산 복잡도 핫스팟 목록
- OOM 위험 패턴 탐지
- 청킹·스트리밍·오프로드 전략 제안

---

### sim-physics-auditor

**트리거 예시:**
- "물리 방정식 검토해줘", "단위 맞는지 확인해줘"
- "시뮬레이션 파라미터 바꿨는데 괜찮아?"
- "C++ → Python 포팅 후 결과가 달라"

**입력:**
```
검토 대상 방정식·수식 및 가정
단위·차원 정보
시뮬레이션 파라미터 (주파수, 속도, 샘플 레이트, 스텝 크기 등)
언어/프레임워크 포팅 정보 (선택)
```

**출력:**
- 단위·차원 검증 결과
- 좌표계 일관성 보고
- 샘플링 이론 준수 여부 (나이퀴스트 등)
- 수치 안정성 경고 (제어 루프, 적분기, 필터)
- 방정식 등가 검증 (포팅 시)

---

### rag-data-quality

**트리거 예시:**
- "RAG 품질 검사해줘", "코퍼스 품질 어때?"
- "리콜이 낮아졌어", "청킹 전략 바꾸려고"
- "새 문서 추가했는데 평가해줘"

**입력:**
```
코퍼스 경로 or 인덱스 통계
청킹 설정 (청크 크기, 오버랩, 분할 기준)
도메인 FAQ/쿼리 목록 (선택)
임베딩 모델 정보 (선택)
```

**출력:**
- 메타데이터 완결성·스키마 일관성 검증
- 청킹 규칙·오버랩 설정 적절성 보고
- 중복·유사 중복 탐지
- 도메인 커버리지 분석
- Recall@k, 리랭킹 효과성 평가

## 타 파이프라인 투입 가이드

이 스킬들은 아래 파이프라인에서 **선택 투입 감사 스킬**로 사용된다.
각 파이프라인 MANUAL.md의 `★ 도메인 특화 감사 투입 시점` 섹션을 함께 참조한다.

---

### core-engineering_pipeline_skills 투입 시점

```
③ uf-designer 완료 (uf.md 확정)
        ↓
   ★ [도메인 해당 시 여기서 투입]
        ↓
④ uf-implementor 실행
```

| 투입 조건 | 스킬 | 목적 |
|-----------|------|------|
| UF 중 GPU/HPC 연산이 있다 | `gpu-hpc-guard` | 구현 전 메모리·복잡도 위험 사전 파악 |
| UF 중 물리 시뮬레이션 방정식이 있다 | `sim-physics-auditor` | 방정식 검증 후 구현 지시 |
| UF 중 RAG 또는 임베딩 파이프라인이 있다 | `rag-data-quality` | 코퍼스 품질 확보 후 구현 |

---

### req_impl_review_pipeline_skills 투입 시점

```
② uf-designer 완료 (uf.md 확정)
        ↓
   ★ [도메인 해당 시 여기서 투입]
        ↓
③ repo-doc-writer 실행
```

| 투입 조건 | 스킬 | 목적 |
|-----------|------|------|
| GPU/HPC 연산 포함 UF가 있다 | `gpu-hpc-guard` | Cursor 구현 지시 전 위험 파악 |
| 물리 시뮬레이션 포함 UF가 있다 | `sim-physics-auditor` | 방정식 오류를 Cursor 구현 전 제거 |
| RAG/임베딩 관련 UF가 있다 | `rag-data-quality` | 코퍼스 품질 확보 후 구현 지시 |

> **추가 투입 시점:** `code-reviewer` 이후 CRITICAL 항목이 도메인 특화 문제(OOM, 단위 오류, 리콜 저하 등)인 경우,
> 해당 스킬로 심층 감사 후 `cursor-task-formatter` (Mode B) fix 프롬프트 생성.

---

## 스킬 선택 기준 요약

```
구현에 GPU·대용량 텐서·배치 연산이 포함된다
  → gpu-hpc-guard

물리 방정식·시뮬레이션 파라미터·다른 언어로 포팅이 포함된다
  → sim-physics-auditor

RAG 코퍼스·임베딩·리트리버 파이프라인이 포함된다
  → rag-data-quality

위 중 둘 이상 해당한다
  → 해당 스킬 모두 순차 투입
```
