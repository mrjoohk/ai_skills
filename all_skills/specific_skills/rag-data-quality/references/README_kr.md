# RAG Data Quality - 한국어 가이드

## 스킬 개요

RAG(검색 증강 생성) 코퍼스의 품질을 종합적으로 평가하는 스킬입니다. 메타데이터 완전성, 청킹 규칙 일관성, 중복 탐지, 도메인별 커버리지를 검증하며, Recall@k와 리랭크 효과성을 측정합니다. 갭 기반의 타겟 수집 계획을 제시하여 코퍼스 품질을 지속적으로 향상시킵니다.

## 언제 사용하는가

다음과 같은 상황에서 사용합니다:
- 신규 문서를 인제스트하기 전/후 품질 검증
- Recall 또는 답변 근거(Grounding) 품질이 저하될 때
- 새로운 도메인이나 주제 영역으로 코퍼스를 확장할 때
- 청킹 전략이나 임베딩 모델을 변경할 때
- 정기적인 코퍼스 감사(Audit) 및 품질 점검

## 입력

다음 정보를 제공해야 합니다:
- **코퍼스 정보:** 코퍼스 경로 또는 인덱스 통계
- **청킹 설정:** 청크 크기, 오버랩, 분리 기준
- **평가 세트:** 질문/답변 쌍 + 인용 출처
- **도메인 힌트:** 기술문서, 법률, 의료, 제품 매뉴얼 등

## 출력물

다음 산출물이 생성됩니다:

| 산출물 | 설명 |
|--------|------|
| **품질 리포트** | 커버리지%, 중복률, 누락 메타데이터 비율 |
| **메타데이터 분석** | 필수 필드(doc_id, source, domain, date, chunk_id, token_count) 검증 |
| **청킹 평가** | 현재 설정 문제점, 개선 제안, Recall 영향 추정 |
| **타겟 수집 계획** | 저커버리지 갭 식별 → 수집 대상 문서 유형 추천 |
| **평가 프로토콜** | Recall@k, MRR, Precision@k 지표 및 임계값 |

## 슬래시 명령 예시

### 예시 1: 메타데이터 감사
```
data/corpus/ 를 스캔하여 각 문서의 필수 메타데이터 필드
(doc_id, source, domain, title, date, chunk_id, token_count) 누락 여부를 검증하라.
누락 필드를 도메인별로 정리하여 reports/rag_quality.md 에 출력하라.
```

### 예시 2: 청킹 전략 평가
```
현재 청킹 설정(크기=512, 오버랩=32)을 평가하라.
표와 섹션 제목 처리 규칙이 올바른지 확인하고,
개선된 청킹 규칙을 configs/chunking.yaml 로 제안하라.
Recall@5에 미치는 영향을 추정하라.
```

### 예시 3: 갭 기반 수집 계획
```
20개의 평가 쿼리에 대해 저커버리지 클러스터를 식별하고,
각 갭을 해소할 수 있는 구체적인 문서 유형을 제안하라.
랜덤 수집 전략 대신 갭 기반 수집 계획을 수립하라.
```

## 다른 스킬과의 연결

**함께 사용하는 스킬 흐름:**

1. **rag-data-quality** → **eval-planner**
   - RAG 품질 평가 후, 평가 데이터셋 계획 수립

2. **rag-data-quality** → **eval-runner**
   - 제안된 청킹 설정으로 Recall 벤치마크 실행

3. **core-engineering** → **rag-data-quality**
   - RAG 파이프라인 설계 후 코퍼스 품질 검증

## 메타데이터 스키마 (최소 필수)

### 문서 수준
```yaml
doc_id:      고유 식별자
source:      원본 파일 경로 또는 URL
domain:      도메인 힌트 (기술문서, 법률, 의료, 제품 매뉴얼 등)
title:       문서 제목
date:        작성/수정일 (ISO 8601)
version:     버전 (있는 경우)
author_org:  저자 또는 조직
```

### 청크 수준
```yaml
chunk_id:    문서 내 고유 청크 식별자
chunk_index: 문서 내 순번
token_count: 청크 토큰 수
```

## 청킹 규칙 가이드

| 규칙 | 설명 |
|------|------|
| **청크 크기** | 목표 토큰 수 (예: 256~512 tokens) |
| **오버랩** | 연속 청크 간 겹치는 토큰 수 (예: 32~64 tokens) |
| **표 보존** | 표(Table) 중간에서 분리하지 않음 |
| **제목 연결** | 섹션 제목을 해당 본문과 함께 유지 |
| **코드 블록** | 코드 블록은 분리하지 않음 |
| **분리 기준** | 문단 경계 우선, 문장 경계 차선 |

## 품질 지표 (Metrics)

| 지표 | 정의 | 목표 임계값 |
|------|------|----------|
| **Coverage** | 평가 쿼리 셋이 코퍼스에서 답변 가능한 비율 | >= 90% |
| **Duplication Rate** | 근사 중복 청크 비율 | <= 5% |
| **Missing Metadata** | 필수 메타데이터 누락 청크 비율 | 0% |
| **Recall@k** | 평가 쿼리에 대해 top-k 내 정답 청크 포함 비율 | >= 85% (k=5) |
| **Rerank Lift** | 리랭크 전후 top-1 정확도 차이 | >= 10% |
| **Grounded Answer Rate** | 인용(Citation) 포함 답변 비율 | >= 95% |

## 타겟 수집 (Targeted Acquisition)

1. **갭 식별:** 평가 쿼리 셋에서 저커버리지 클러스터 찾기
2. **문서 추천:** 각 갭에 대응하는 구체적 문서 유형 제시
3. **획득 우선순위:** 커버리지 개선 효과 높은 항목부터 수집
