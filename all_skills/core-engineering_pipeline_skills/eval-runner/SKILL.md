# Eval Runner

Takes `evaluation_plan.md` or a metric list as input,
**generates calculation functions**, and outputs experiment results in a **readable report format**.
This is the execution part of Stage 8, continuing from the output of `eval-planner`.

---

## When to Use
- When `evaluation_plan.md` generated by `eval-planner` is available (recommended)
- When metric names and data paths are provided directly (works without evaluation_plan)
- When you want to organize multiple experiment results into a comparison table
- When adding metric calculation scripts to a CI pipeline

---

## Inputs
- `evaluation_plan.md` (output from eval-planner) — or provide metric names + thresholds directly
- Experiment result data: file paths, arrays, or numeric values entered directly
- List of experiments to compare (optional: for multi-experiment comparison)

---

## Execution Steps

### Step 1 — Parse Inputs
- If `evaluation_plan.md` exists, read it to extract metric, threshold, and dataset information
- Otherwise, proceed with metric names and data provided by the user
- Detect domain and select calculation library (see `reference.md`)

### Step 2 — Design Calculation Functions
Design calculation functions for each metric:

```python
# Example function structure
def compute_<metric>(prediction, reference, **kwargs):
    """
    Args:
        prediction: <type, shape, unit>
        reference:  <type, shape, unit>
    Returns:
        score: float (<unit>, ↑ higher is better)
    """
    ...
    return score
```

Design principles:
- One function per metric (single responsibility)
- Specify I/O types and units in docstring
- Support both batch and single-sample processing
- Minimize library dependencies and specify versions

### Step 3 — Aggregate Results and Generate Report
Write calculation results to a report following the Output Template below.

---

## Output Template

### Results Report (`reports/eval/<task_name>_<timestamp>.md`)

```markdown
# Evaluation Report: <task_name>
Date: <date> | Commit: <commit_sha>

## Results Summary

| Metric | <Exp-01 Baseline> | <Exp-02 Proposed> | Delta | Target | Status |
|---|---:|---:|---:|---:|:---:|
| SI-SDR (↑) | 9.2 dB | 12.8 dB | +3.6 | ≥ 12.0 | ✅ PASS |
| PESQ (↑)   | 2.3    | 2.9     | +0.6 | ≥ 2.8  | ✅ PASS |
| WER (↓)    | 18.2%  | 14.1%   | -4.1 | ≤ 15%  | ✅ PASS |

## Primary Metric: SI-SDR
- **Result:** 12.8 dB ✅ (Target: ≥ 12.0 dB)
- **Improvement:** +3.6 dB vs Baseline (+39.1%)

## Acceptance Criteria Judgment
Given <test conditions>, When <model inference runs>,
Then SI-SDR >= 12.0 dB → **PASS** (measured: 12.8 dB)

## Evidence
- Calculation script: `scripts/eval/<task_name>_eval.py`
- Raw results: `evidence_pack/metrics.yaml`
- Dataset: <dataset_name> (N=<sample_count>)
```

---

## Calculation Script Output Locations
- `scripts/eval/<task_name>_eval.py` — Reusable collection of calculation functions
- `reports/eval/<task_name>_<timestamp>.md` — Report
- `evidence_pack/metrics.yaml` — Numeric records for CI integration

---

## Notes
- Always verify that metric direction (↑/↓) and threshold comparison direction match
- When comparing multiple experiments, specify that identical datasets, splits, and random seeds were used
- Report p-value or bootstrap CI when statistical significance is needed
- Calculation library & version reference: `references/reference.md`
