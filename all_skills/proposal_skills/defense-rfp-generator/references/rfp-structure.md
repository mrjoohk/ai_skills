# 방위사업청 형식 합성 RFP 구조 템플릿

방위사업청 연구개발 과제 공모 양식에서 흔히 볼 수 있는 7개 섹션 구조.
defense-rfp-generator 스킬이 이 구조를 채워 합성 RFP를 생성한다.

---

## 문서 표지

```
[기관 로고 자리]

연구개발 과제 공모 요청서 (RFP)
Request for Proposal — Research & Development

과 제 명: [과제명 전체]
과제 분야: [기술 도메인 코드]
접수 기간: [YYYY.MM.DD ~ YYYY.MM.DD] (실제 날짜 미정 → "추후 공고" 기재)
문서 버전: 합성본 v1.0 (유사 프로그램 분석 기반)
```

---

## 1섹션. 사업 개요 (Program Overview)

### 1.1 사업 배경 및 필요성
- 현재 군 운용 상의 문제점 / 기술적 공백 (2~3 문단)
- 국내외 기술 동향 및 위협 환경 변화 (1~2 문단)
- 본 과제의 전략적 필요성 (1 문단)

### 1.2 사업 목적
- 핵심 목적 서술 (1문장 요약)
- 세부 목적 3~5개 (번호 목록)

### 1.3 사업 규모
| 항목 | 내용 |
|------|------|
| 연구 기간 | N년 (YYYY.MM ~ YYYY.MM) |
| 총 예산 | 약 XX억 원 |
| 연구 형태 | 기초연구 / 응용연구 / 개발연구 |
| 주관 기관 | 대학 / 출연연 / 기업 (복수 신청 가능) |
| 보안 등급 | 일반 / 대외비 (합성 RFP는 일반으로 기재) |

## 5섹션. 평가 기준 (Evaluation Criteria)

### 5.1 평가 체계 개요

총 100점 만점의 기술·사업성 통합 평가.

### 5.2 평가 항목 및 배점 (E-1 배점 표)

| 평가 영역 | 평가 항목 | 배점 |
|-----------|-----------|------|
| **기술성 (65점)** | 기술개발 계획의 구체성 | 20 |
| | 연구팀 역량 및 연구 인프라 | 15 |
| | 기술적 접근 방법의 타당성 | 15 |
| | 혁신성·도전성 | 15 |
| **사업성 (35점)** | 예산 계획의 적절성 및 효율성 | 15 |
| | 추진 일정의 현실성 | 10 |
| | 성과 활용 및 파급 효과 | 10 |
| **합계** | | **100** |

### 5.3 전략적 시사점 (암묵적 우선순위)

아래 항목은 배점 외에 심사위원이 중점적으로 살펴보는 요소다:
- **혁신성**: 기존 국내 기술과 차별화되는 기술적 도약점 명시 여부
- **실현 가능성**: TRL 목표가 기간/예산 대비 현실적인지
- **활용 계획**: 연구 완료 후 실 체계 연동 또는 기술이전 방안의 구체성

## 6섹션. 제출 요건 (Submission Requirements)

### 6.1 제안서 구성
- 과제 개요서 (A4 10페이지 이내)
- 세부 연구 계획서 (A4 50페이지 이내)
- 예산 계획서 (별도 양식)
- 연구팀 구성 및 역량 증빙자료

### 6.2 제안서 작성 지침
- 한국어 기준 작성 (영문 병기 허용)
- 폰트: 한글은 맑은고딕 10pt 이상, 영문 Arial 10pt 이상
- 기밀 정보 포함 불가 (일반 문서 기준)

---

## 7섹션. 기타 사항

### 7.1 지식재산권
- 연구 결과물의 지식재산권은 주관 기관과 발주 기관이 협의하여 결정
- 특허 출원 시 연구비 지원 가능

### 7.2 보안 및 기밀 관리
- 연구 과정에서 획득한 군사 관련 정보는 보안 규정에 따라 관리
- 연구 결과 공개 전 주관 기관 검토 필요

### 7.3 유의 사항
- 본 문서는 유사 프로그램 분석 기반의 합성 RFP임
- 실제 공모 시 공식 문서로 교체 필요
- 합성 RFP 기반 작성된 제안서는 defense-proposal-prep 스킬로 재분석 권장
