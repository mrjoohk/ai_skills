# Defense KPI Benchmark 스킬

방산 제안 기술의 **구성 기술별 국내외 연구동향을 조사**하여 현재 세계 최고 수준(State-of-the-Art)을
수치로 파악하고, 이를 바탕으로 **근거 있는 KPI 목표치**를 도출한다.

성능 목표치가 "합리적으로 설정되었다"는 것을 심사위원에게 설득하려면
**목표의 근거**가 있어야 한다. 이 스킬이 그 근거를 만든다.

> 📖 **참조 파일**:
> - `references/benchmark-survey-template.md` — 기술별 성능 조사 카드 양식
> - `references/kpi-setting-guide.md` — KPI 목표치 설정 방법론 및 근거 작성법

---

## 스킬 위치 (전체 워크플로에서)

```
defense-proposal-prep (Step 2: 기능 도출 + TRL 갭 분석)
    ↓
defense-kpi-benchmark  ← 여기
    ↓
defense-proposal-overview (3항: 연구 목표 및 필요성 — 정량 목표표)
defense-proposal-rd-plan  (2항: 연구개발 목표 및 내용)
defense-proposal-tech-analysis (4~5항: 소요기술 분석)
```

**입력**: `defense-proposal-prep` 결과의 구성 기술 목록 (또는 사용자가 직접 제시)
**출력**: KPI 목표표 + 기술별 성능 벤치마크 카드 (MD) + KPI 요약 슬라이드 (PPTX)

## Step 1 — 조사 대상 기술 및 KPI 후보 확정

### 1-1. 구성 기술 목록 수집

`defense-proposal-prep` 결과에서 미확보 기술 목록과 TRL 갭 분석표를 가져온다.
없으면 사용자에게 "어떤 기술들로 구성되나요?" 질문하여 목록화한다.

각 기술에 대해 아래를 파악:
- 기술명 (구체적일수록 좋음 — "AI"보다 "CFAR 기반 표적 탐지 알고리즘")
- 기술 도메인 (신호처리 / 유도제어 / 소재 / 통신 / AI·SW 등)
- 최종 적용 환경 (운용 조건: 온도범위, 전자기 환경, 플랫폼 등)

### 1-2. 기술별 KPI 후보 도출

각 기술에 대해 **"이 기술의 성능을 측정하는 핵심 지표가 무엇인가"**를 먼저 정의한다.

`references/kpi-setting-guide.md`의 "도메인별 KPI 사전"을 참조하여 후보를 선정.
후보가 너무 많으면 아래 기준으로 3~5개로 압축:

| 선택 기준 | 설명 |
|---|---|
| 측정 가능성 | 실험/시험으로 검증 가능한가? |
| 차별화 가능성 | 기존 대비 우리가 개선할 수 있는 지표인가? |
| 발주처 관심도 | RFP 또는 군 요구사항에 언급된 지표인가? |
| 혁신성 표현력 | 이 지표의 향상이 운용상 효과로 직결되는가? |

## Step 2 — 국내외 연구동향 및 SoA 조사

각 구성 기술별로 웹 검색을 수행하여 현재 세계 최고 수준(State-of-the-Art)을 파악한다.

### 2-1. 조사 방법

**검색 쿼리 패턴 (기술별로 반복):**
```
[기술명] state-of-the-art performance [연도범위: 2022-2025]
[기술명] benchmark [핵심 KPI 지표명]
[기술명] DARPA / NATO / IEEE 최신 결과
[기술명] 국내 연구 현황 방위산업기술지원센터
[기술명] 최신 논문 성능 비교
```

**조사 우선순위:**
1. 국방 관련 기관 발표 (DARPA, DSTL, 방산기술지원센터, ADD 등)
2. 상위 학회/저널 논문 (IEEE Trans., AIAA, ACM 등)
3. 정부 R&D 보고서 (ITAR 비제한 공개 자료)
4. 산업계 백서 및 기술 발표

### 2-2. 조사 결과 기록 양식

`references/benchmark-survey-template.md`의 "기술 성능 조사 카드" 양식으로 기록.

각 기술별 카드 내용:
```
[기술명]
───────────────────────────────────
해외 SoA (최고 수준):
  · 기관/논문: [이름], [연도]
  · 달성 성능: [KPI1] = X [단위], [KPI2] = Y [단위]
  · 환경 조건: [조건 명시]
  · 출처 URL / DOI: [링크]

해외 평균 수준 (주요 참여국):
  · [국가/기관]: [KPI1] = X, [KPI2] = Y (출처: [])
  · [국가/기관]: [KPI1] = X, [KPI2] = Y (출처: [])

국내 SoA:
  · 기관/과제: [이름], [연도]
  · 달성 성능: [KPI1] = X, [KPI2] = Y
  · 출처: [보고서/논문]

현재 기술 한계:
  · [한계 1]: 원인 — [물리적/기술적 근거]
  · [한계 2]: 원인 — [근거]

기술 발전 트렌드:
  · 최근 3년 성능 향상 추이: [KPI] 기준 연 N% 향상
  · 주요 접근 방향: [방법론 1], [방법론 2]
───────────────────────────────────
```

## Step 3 — KPI 목표치 설정

조사 결과를 바탕으로 각 KPI의 목표치를 설정한다.
목표치 설정은 **도전적이되 실현 가능**해야 하며, 반드시 **근거**를 함께 제시한다.

### 3-1. 목표치 설정 방법론 (3가지 중 선택)

`references/kpi-setting-guide.md`의 상세 가이드를 참조.

**방법 A: SoA 기반 목표 (가장 일반적)**
- 국내외 SoA 대비 N% 향상을 목표로 설정
- 근거: "현재 최고 수준 X를 N% 개선한 Y를 목표로 하며, 이는 [접근법]으로 달성 가능"
- 적합: 기술이 명확하고 비교 가능한 벤치마크가 존재할 때

**방법 B: 운용 요구사항 역산 (가장 설득력 높음)**
- 최종 운용 성능(군 요구사항)에서 역산하여 구성 기술 목표치 도출
- 근거: "체계 요구사항 Z 달성을 위해 이 기술은 최소 Y 수준이 필요 (시스템 해석 근거)"
- 적합: RFP에 운용 요구사항이 명확히 제시된 경우

**방법 C: 기술 발전 트렌드 외삽**
- 현재 트렌드를 연구 기간(N년) 후까지 외삽하여 달성 가능 수준 예측
- 근거: "최근 3년 연 X% 향상 트렌드 + 우리의 핵심 기술 적용 시 추가 Y% 개선 예상"
- 적합: 발전 속도가 빠른 AI/SW 분야

### 3-2. 목표치의 도전성 검증

설정한 목표치가 "적절히 도전적"인지 아래 기준으로 검증:

```
너무 쉬움 (❌): 현재 국내 수준과 동일하거나 해외 평균 이하
적당함    (✅): 현재 국내 SoA를 넘고, 해외 SoA에 근접하거나 동등
도전적    (✅): 현재 해외 SoA를 넘어서는 수준 (단, 근거 필수)
불가능    (❌): 물리적 한계를 초과하거나 기간 내 TRL 달성 불가
```

목표치가 "도전적" 영역에 들어가면 반드시 달성 가능 근거를 추가:
- 우리만의 접근법 (핵심 알고리즘, 소재, 구조)
- 유사 기술에서의 선례
- 핵심 인력의 사전 연구 실적

### 3-3. 정량 목표 vs 정성 목표 분류

| 목표 유형 | 정의 | 표현 방식 |
|---|---|---|
| **정량 목표** | 수치로 측정·검증 가능 | "탐지거리 ≥ 50 km (SNR 15 dB 조건)" |
| **정성 목표** | 기능/특성으로 표현, 세부 기준은 추후 정의 | "악천후 환경에서의 안정적 동작 능력 확보" |

정성 목표도 가능하면 세부 합격 기준(Pass/Fail 조건)을 달아서 반증 가능하게 만든다.

## Step 4 — KPI 목표표 생성

### 4-1. 최종 KPI 목표표 (제안서 직접 삽입용)

| KPI 항목 | 단위 | 현재 국내 SoA | 현재 해외 SoA | 목표치 | 향상률 | 설정 근거 | 검증 방법 |
|---|---|---|---|---|---|---|---|
| [KPI 1] | [단위] | X | Y | Z | +(Z-Y)/Y% | [방법 A/B/C] | [시험 방법] |
| [KPI 2] | | | | | | | |

**표 작성 규칙:**
- 목표치는 "≥ Z [단위]" 또는 "≤ Z [단위]" 형식으로 방향성 명시
- 측정 조건을 반드시 괄호 안에 병기 (예: "SNR 10 dB, 0.5 m 파고 조건")
- 출처가 있는 수치는 각주 형식으로 참고문헌 번호 표시

### 4-2. 기술-KPI 연결 매트릭스

어떤 구성 기술이 어떤 KPI에 영향을 미치는지 매핑:

| KPI 항목 | 영향 기술 1 | 영향 기술 2 | 영향 기술 3 | 지배적 기술 |
|---|---|---|---|---|
| [KPI 1] | T1 (기여 60%) | T3 (기여 30%) | T5 (기여 10%) | T1 |

→ 지배적 기술의 개발이 해당 KPI 달성에 직결됨을 제안서에서 강조.

## Step 5 — 혁신성·도전성 연결

KPI 분석 결과를 `defense-proposal-prep`의 혁신성 포지셔닝과 연결한다.

### 5-1. 혁신성 수치 근거 생성

```
[혁신성 표현 예시]
"본 연구의 핵심 KPI인 [KPI 항목]은 현재 해외 SoA([출처])의 Y [단위] 대비
Z [단위]를 목표로 하며, 이는 N% 향상에 해당한다.
이는 [핵심 기술 A]와 [핵심 기술 B]의 융합 접근으로 달성 가능하며,
유사 분야인 [참조 분야]에서 이미 M% 향상이 보고된 바 있다 [출처]."
```

### 5-2. 도전성 수치 근거 생성

```
[도전성 표현 예시]
"목표 성능 Z [단위]는 현재 국내 최고 수준(X) 대비 K% 향상을 요구하며,
이는 단순 기존 기술의 최적화로는 달성 불가능한 수준이다.
이를 위해 [핵심 기술적 난제]를 해결해야 하며,
본 연구는 [독창적 접근법]으로 이를 극복한다."
```

## Step 6 — 결과물 생성

### 6-1. 벤치마크 분석 MD 문서 저장

**파일명**: `YYMMDD_HHMM_kpi_benchmark_[과제명약어].md`

포함 내용:
- 기술별 성능 조사 카드 (전체)
- KPI 목표표
- 기술-KPI 연결 매트릭스
- 혁신성·도전성 수치 근거 문구
- 참고문헌 목록 (출처 전체)
- 판단 근거 요약

### 6-2. KPI 요약 PPTX 슬라이드 생성

`pptx 스킬`로 다음 슬라이드 생성:

**슬라이드 1: 국내외 기술 수준 비교**
- 기술별 현재 SoA 비교 표 또는 레이더 차트
- 국내/해외/목표 3단 비교 구조

**슬라이드 2: KPI 목표표**
- 최종 KPI 목표표 (표 전체)
- 우측 여백에 "근거 요약" 텍스트 박스

**슬라이드 3: 기술-KPI 연결 매트릭스**
- 히트맵 또는 매트릭스 표
- 지배적 기술 강조 (ACCENT 색상)

**슬라이드 4: 혁신성·도전성 수치 근거**
- 핵심 KPI 1~2개 선택하여 "현재 SoA vs 목표" 비교 바 차트
- 4막 혁신 서사 텍스트 요약

### 6-3. 다운스트림 스킬 연결 포인트

## 작업 규칙

- **출처 필수**: 모든 수치 데이터는 출처(기관, 논문, 발표, 연도) 명시. "알려진 바에 따르면" 같은 표현 금지.
- **조건 명시**: 성능 수치는 반드시 측정 조건(환경, 입력, 기준)을 병기.
- **보수적 SoA 인용**: 과장된 수치보다 검증된 공개 자료 우선. 주장하기 어려운 수치는 "추정" 표시.
- **목표치 일관성**: 이 스킬에서 확정한 KPI 수치는 이후 모든 제안서 섹션에서 동일하게 사용.
- **GLOBAL_RULES 준수**: Rule 1(승인 후 출력), Rule 2(MD 저장), Rule 3(판단 근거), Rule 4(파일 로그).
