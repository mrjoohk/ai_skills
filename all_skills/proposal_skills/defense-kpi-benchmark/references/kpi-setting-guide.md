# KPI 목표치 설정 가이드

방산 연구 제안서의 성능 목표치(KPI)를 설정하는 방법론.
"왜 이 수치를 목표로 삼았는가"에 대한 설득력 있는 근거를 만드는 것이 핵심이다.

---

## 1. KPI 설정의 3원칙

### 원칙 1: 측정 가능성 (Measurability)
KPI는 반드시 시험/실험으로 검증할 수 있어야 한다.
- ✅ "탐지거리 ≥ 50 km (RCS 1 m², SNR ≥ 10 dB 조건)"
- ❌ "우수한 탐지 성능 확보"
- ❌ "세계 최고 수준의 정확도"

### 원칙 2: 도전적 실현 가능성 (Ambitious but Achievable)
목표치는 "도전적이지만 근거 있게 달성 가능"한 수준이어야 한다.
- 너무 쉬운 목표: 심사위원이 도전성 없다고 평가 → 배점 손실
- 불가능한 목표: 실현 가능성 의심 → 제안 신뢰도 하락
- 적절한 범위: **현재 국내 SoA + (10~40%)** 또는 **해외 SoA 수준 근접**

### 원칙 3: 인과 연결성 (Causality)
KPI 목표치와 "우리가 개발할 핵심 기술" 사이의 인과관계가 명확해야 한다.
- 이 기술이 어떻게 이 KPI를 향상시키는가?
- 다른 기술로는 왜 이 목표를 달성하기 어려운가?

## 2. 목표치 설정 방법론 상세

### 방법 A: SoA 기반 목표 설정

**절차:**
1. 국내 SoA 수치 확인 (출처 포함)
2. 해외 SoA 수치 확인 (출처 포함)
3. 두 수치 사이 또는 해외 SoA 수준을 목표로 설정
4. 달성 근거: 핵심 기술 접근법 + 유사 사례

**목표치 설정 범위 가이드:**
```
해외 SoA가 존재하는 경우:
  공격적: 해외 SoA + 10~20% (세계 최고 수준 추월 목표)
  표준:   해외 SoA ± 0~10%  (해외 수준 추격)
  보수적: 국내 SoA + 20~30% (국내 최고 수준 개선)

해외 SoA가 없거나 비공개인 경우:
  국내 SoA 기준 30~50% 향상 목표 + 기술적 근거
```

**근거 작성 템플릿:**
```
"[KPI 항목]의 목표치 [Z 단위]는 현재 국내 최고 수준([X 단위], [출처])
 대비 [N%] 향상된 수치이며, 해외 SoA([Y 단위], [출처])에 근접한 수준이다.
 이는 본 연구의 핵심 기술인 [기술명]을 통해 달성 가능하며,
 유사 방법론으로 [M%] 향상이 보고된 바 있다 ([참조 논문])."
```

---

### 방법 B: 운용 요구사항 역산 (Top-Down)

군 운용 요구사항(ROC)이 제시된 경우 가장 설득력 있는 방법.

**절차:**
1. 체계 수준 운용 성능 요구사항 확인 (RFP 또는 ROC)
2. 시스템 해석/예산 분배(Budget Analysis)로 구성 기술 목표 역산
3. 역산된 목표치를 현재 기술 수준과 비교하여 갭 확인
4. 갭이 너무 크면 단계적 목표(1단계 → 2단계) 설정

**역산 예시 (레이더 탐지거리 → 신호처리 성능):**
```
체계 요구: 탐지거리 ≥ 100 km (RCS 0.1 m²)
  ↓ 레이더 방정식 역산
안테나 이득: G ≥ 35 dBi (주어진 파라미터)
  ↓ 시스템 SNR 예산
신호처리 이득: SNR gain ≥ 20 dB 필요
  ↓ 구성 기술 목표
CFAR 알고리즘: Pfa ≤ 10⁻⁶ @ SNR 15 dB 조건에서 Pd ≥ 0.9
```

**근거 작성 템플릿:**
```
"체계 요구 성능([운용 요구사항])을 달성하기 위해 [시스템 해석 방법]을 통해
 역산한 결과, [구성 기술]은 최소 [Z 단위] 수준을 만족해야 함을 확인하였다.
 (상세 계산 내용: 별첨 N 참조)"
```

---

### 방법 C: 기술 발전 트렌드 외삽

AI/SW 분야처럼 기술 발전이 빠른 분야에 적합.

**절차:**
1. 최근 3~5년 해당 KPI의 연도별 최고 수준 데이터 수집
2. 선형 또는 지수 트렌드 피팅
3. 연구 완료 시점(N년 후)으로 외삽하여 달성 가능 수준 추정
4. 우리의 핵심 기술이 트렌드 대비 추가 개선을 제공하는 근거 제시

**외삽 + 가속 효과 템플릿:**
```
"해당 기술 분야의 최근 [N]년간 추이를 분석하면
 [KPI]는 연 평균 [X%] 향상되어 왔다 (그림 N 참조).
 이 트렌드를 연구 완료 시점([년도])으로 외삽하면
 [Y 단위] 수준이 예상되며, 본 연구의 [핵심 기술] 적용 시
 트렌드 대비 추가 [Z%] 향상이 가능하여 최종 목표 [W 단위]를 설정한다."
```

## 3. 도메인별 KPI 사전

방산 분야별 대표적인 성능 지표 모음. 스킬 사용 시 이 목록에서 해당 도메인 KPI를 선택한다.

### 🔭 레이더 / 탐지
| KPI | 단위 | 측정 조건 명시 필요 항목 |
|---|---|---|
| 탐지거리 (Pd) | km | RCS, SNR, Pd 기준, 클러터 환경 |
| 탐지 확률 | % | Pfa, SNR, 표적 종류 |
| 허위 경보율 (Pfa) | /시간 또는 무차원 | 클러터 분포 모델 |
| 거리 해상도 | m | 대역폭, 파형 |
| 각도 해상도 | deg 또는 mrad | 안테나 구경 |
| 재방문 주기 | sec | 스캔 모드 |
| ECCM 성능 | dB (재밍 마진) | 재밍 유형 |
| 처리 지연 (latency) | ms | 처리 아키텍처 |

### 🚀 유도 / 항법
| KPI | 단위 | 조건 |
|---|---|---|
| 명중 정확도 (CEP) | m | 사거리, 표적 종류, 기상 조건 |
| 탐색기 시야각 (FOV) | deg | |
| 최대 사거리 | km | 비행 프로파일 |
| 최소 교전 거리 | km | 기동 여유 |
| 항법 정확도 (INS) | m (위치), m/s (속도) | GPS 차단 환경 |
| 항재밍 마진 | dB | GPS 재밍 시나리오 |

### 📡 통신 / 전자전
| KPI | 단위 | 조건 |
|---|---|---|
| 데이터 전송률 | Mbps | 링크 거리, 안테나 이득 |
| 통신 거리 | km | 지형, 주파수대 |
| 항재밍 성능 | dB | 재밍 유형, J/S |
| 스펙트럼 탐색 속도 | GHz/s | 주파수 범위 |
| 신호 분류 정확도 | % | 신호 유형 수, SNR |
| 반응 시간 (EA) | ms | 위협 유형 |

### 🤖 AI / 자율 시스템
| KPI | 단위 | 조건 |
|---|---|---|
| 표적 인식 정확도 | % (mAP, F1) | 데이터셋, 클래스 수, 조건 |
| 처리 속도 (inference) | FPS 또는 ms | 하드웨어 플랫폼 |
| 모델 크기 | MB 또는 파라미터 수 | 압축 방법 |
| 전력 소모 | W | 동작 조건 |
| 강건성 (adversarial) | % 정확도 유지율 | 공격 유형, 강도 |
| OOD 탐지율 | % | 분포 외 데이터 비율 |

### ⚙️ 기계 / 추진 / 구조
| KPI | 단위 | 조건 |
|---|---|---|
| 추력 대 중량비 (T/W) | 무차원 | 운용 고도, 속도 |
| 비추력 (Isp) | sec | 연료 종류 |
| 구조 중량 | kg 또는 kg/m² | 설계 하중 조건 |
| 수명 (MTBF) | 시간 | 운용 환경 |
| 진동 내구성 | g (진동 가속도) | 주파수 범위 |
| 열 관리 (최대 표면온도) | °C | 운용 시간 |

### 🔬 소재 / 스텔스
| KPI | 단위 | 조건 |
|---|---|---|
| RCS 감소량 | dBsm | 주파수, 입사각 |
| 적외선 방사율 | 무차원 | 파장대, 온도 |
| 전자파 차폐 효율 | dB | 주파수 범위 |
| 강도-대-중량비 | MPa/(kg/m³) | 시험 표준 |
| 내열 온도 | °C | 노출 시간 |

## 4. 목표치 설정 후 필수 검증 항목

설정된 모든 KPI에 대해 아래를 확인:

### 내부 일관성 검증
- [ ] KPI 간 상충(trade-off) 관계가 존재하는가? (예: 탐지거리 vs 해상도 vs 처리속도)
  → 상충이 있으면 "trade-off 곡선" 또는 "설계 점(Design Point)" 명시
- [ ] 모든 KPI를 동시에 만족하는 설계 공간이 존재하는가?

### 외부 정합성 검증
- [ ] RFP의 운용 요구사항과 정합되는가?
- [ ] 연구기간(N년) 내에 TRL 목표와 함께 달성 가능한가?
- [ ] 예산 범위 내에서 필요한 시험·검증 환경을 구축 가능한가?

### 혁신성 관점 검증
- [ ] 목표치가 현재 국내 SoA를 유의미하게 초과하는가? (최소 10% 이상)
- [ ] 해외 SoA와 비교했을 때 "따라잡기" 수준인가, "추월" 수준인가?
- [ ] 목표 달성이 제안 기술 없이는 불가능한가? (기술-KPI 인과 관계)

## 5. 정량/정성 목표 혼합 설계

모든 KPI를 정량화하기 어려운 경우, 정량과 정성을 적절히 혼합한다.

### 정성 목표 작성 원칙
정성 목표도 "검증 가능한 기준"을 붙이면 준정량 목표가 된다:

| 정성 목표 원안 | 개선된 준정량 목표 |
|---|---|
| "강건한 동작 능력" | "기온 -40°C ~ +70°C 범위에서 성능 저하 10% 이내" |
| "신뢰성 있는 통신" | "링크 가용성 99.5% 이상 (30분 기준 시험)" |
| "사용자 친화적 인터페이스" | "훈련받지 않은 운용자의 핵심 기능 조작 시간 ≤ 2분" |
| "실시간 처리 능력" | "센서 입력부터 출력까지 end-to-end 지연 ≤ 100ms" |
