# 실행 매뉴얼 Docx 구조 템플릿

## 문서 전체 구성

```
[표지]
  - 문서 제목: 방산 연구 제안서 스킬 수트 실행 매뉴얼
  - 부제: [과제명]
  - 날짜, 소속 (있으면)

[목차] — TableOfContents (자동 생성)

1장. 본 매뉴얼 개요
  1.1 스킬 수트 구성 표 (9개 스킬 × 단계/스킬명/목적/산출물)
  1.2 전제 조건

2장. 유사 프로그램 조사 요약
  2.1 해외 주요 프로그램 (프로그램별 상세 표)
  2.2 유사 프로그램 핵심 KPI 비교 표

3장. 스킬 실행 절차
  3.1 ~ 3.9 각 스텝 (STEP 1~9)
    - 스텝 카드 (INPUT/OUTPUT 표)
    - 이 과제에서 입력할 내용
    - 주제별 샘플 콘텐츠 (서사, 기술 목록, WBS 등)

4장. 공통 운영 지침
  4.1 파일 관리 규칙 (명명 규칙 표)
  4.2 스킬 실행 전 필수 확인 사항
  4.3 자주 발생하는 오류와 대처

5장. 빠른 참조
  5.1 스킬 호출 명령어 모음 표
  5.2 KPI 목표치 요약 표
```

## 스텝 카드 구조 (각 3.x 섹션)

각 스텝 카드는 3행 테이블:
- **행 1 (헤더)**: PRIMARY 배경 | `STEP N  스킬명 — 목적`
- **행 2 (서브헤더)**: SECONDARY 배경 | `공백 | INPUT (입력) | OUTPUT (출력)`
- **행 3 (콘텐츠)**: LIGHT_BLUE 배경(INPUT) / LIGHT_GREEN 배경(OUTPUT)
- **행 4 (명령어)**: 연회색 배경 | `▶ 호출 방법: "..."`

## 유사 프로그램 표 구조 (2장)

각 프로그램: 2열 표 (항목 / 내용)
항목: 프로그램명, 주관 기관, 기간, 핵심 목표, 주요 성과, 달성 KPI, 학습환경(해당시), 참고 자료

## KPI 비교 표 구조 (2.2 및 5.2)

5열: KPI 항목 | 단위 | 국내 SoA | 해외 SoA | 제안 목표치
- 헤더: PRIMARY 배경
- 제안 목표치 열: ACCENT 색 텍스트 + LIGHT_GOLD 배경

## 기확보/미확보 표 구조 (3.5 섹션)

4열: 구분 | 기술명 | TRL 현재 | 비고
- 기확보 행: LIGHT_GREEN 배경
- 미확보 행: LIGHT_GOLD 배경

## 혁신성 4막 서사 표 (3.4 섹션)

2열: 막 | 내용
행: 1막[현황], 2막[한계], 3막[제안], 4막[효과]
교번 배경 (WHITE / LIGHT_GRAY)

## 페이지 설정

- 용지: A4 (11906 × 16838 DXA)
- 여백: 상하좌우 1134 DXA (약 2cm)
- 기본 폰트: Arial, 11pt (22 half-points)
- 헤더: 우측 정렬, 문서 제목 | 과제명
- 푸터: 중앙, 페이지 번호
