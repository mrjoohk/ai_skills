# Defense Proposal — 연구 계획 스킬

방산 연구 제안서의 **연구 계획** 섹션(6개 항목)을 작성한다.
이 섹션은 "언제, 얼마나, 어떻게 수행하고, 성공 여부를 어떻게 판단할 것인가"를 제시한다.

## 섹션 구성

| 항 | 제목 | 핵심 내용 |
|---|---|---|
| 1 | 연구기간 및 소요예산 | 전체 기간, 연차별/단계별 예산 |
| 2 | 추진 일정 계획 | 간트차트 수준 상세 일정 |
| 3 | 성과평가 계획 | KPI, 평가 방법, 평가 시점 |
| 4 | 기술적 접근방법 및 수행방안 | 핵심 기술 해결 전략 |
| 5 | 연구개발업무 추진체계 | 조직, 역할, 협력 체계 |
| 6 | 세부과제 연구개발계획 | 세부과제별 상세 계획 |

---

## Step 1 — 기본 정보 확인

- 총 연구기간 (예: 3년, 5년)
- 총 예산 규모 (억 원 단위)
- 단계 구분 여부 (1단계 / 2단계, 연차 구분)
- 주관기관 + 참여기관 수 및 명칭
- 세부과제 수 및 명칭

---

## Step 2 — 항목별 콘텐츠 작성

### 2-1. 연구기간 및 소요예산 (1항)

**연구기간 표:**
| 구분 | 기간 | 주요 내용 |
|---|---|---|
| 1단계 | YYYY.MM ~ YYYY.MM | [목표 요약] |
| 2단계 | YYYY.MM ~ YYYY.MM | [목표 요약] |

**소요예산 표 (연차별):**
| 구분 | 1차년도 | 2차년도 | 3차년도 | 합계 |
|---|---|---|---|---|
| 인건비 | | | | |
| 직접비 | | | | |
| 간접비 | | | | |
| **소계** | | | | |

> 예산 세부는 "세부 연구 계획" 섹션(defense-proposal-detail-plan 스킬)에서 더 상세히 작성.

**슬라이드**: 기간 타임라인 + 예산 요약 표 (1–2장)

### 2-2. 추진 일정 계획 (2항)

**defense-wbs-tbs 스킬**을 활용하여 간트차트 형식으로 생성.

추진 일정 내용:
- WBS 기반 주요 활동
- 분기 단위 (Q1/Q2/Q3/Q4) 또는 월 단위
- 마일스톤 표시 (◆ 기호 사용)
- 중간 검토/시험 시점

**마일스톤 목록:**
| 마일스톤 | 시점 | 기준 (Pass/Fail 조건) |
|---|---|---|
| 개념 설계 완료 | 1차년 Q4 | [기준] |
| 시제품 1차 제작 | 2차년 Q2 | [기준] |

**슬라이드**: 간트차트 (1–2장) + 마일스톤 표 (1장)

### 2-3. 성과평가 계획 (3항)

**평가 체계:**
- 최종 목표 성능 지표 (KPI) 목록
- 각 KPI별 평가 방법, 평가 시점, 합격 기준

**성과평가 계획표:**
| KPI 항목 | 목표값 | 평가 방법 | 평가 시점 | 합격 기준 |
|---|---|---|---|---|
| [성능 항목 1] | X [단위] 이상 | [시험 방법] | 2차년 Q4 | ≥ X [단위] |

**평가 기관 및 절차:**
- 내부 평가 vs 외부 평가
- 정부(방위사업청 등) 성과 점검 시점

**슬라이드**: KPI 표 (1장) + 평가 절차 플로우 (1장)

### 2-4. 기술적 접근방법 및 수행방안 (4항)

각 핵심 기술 또는 연구 내용별로:
- **문제 정의**: 해결해야 할 기술적 난제
- **접근 방법**: 알고리즘, 설계 방법, 시험 방법
- **검증 전략**: 시뮬레이션 → 하드웨어-인-더-루프 → 실환경 시험

**defense-diagram 스킬**로 기술 접근 방법 플로우차트(F) 또는 블록다이어그램(B) 생성.

**슬라이드**: 기술 접근 방법론 개요 (1장) + 핵심 기술별 방안 (2–3장)

### 2-5. 연구개발업무 추진체계 (5항)

**조직 구조:**
- 주관기관 역할 (PM, 핵심 기술)
- 참여기관별 역할 및 분담 내용
- 자문위원회 / 기술 감리 구조

**defense-diagram 스킬**로 조직도 (C) 시스템 구성도 유형 사용:
- 주관기관: COLOR_PRIMARY 박스
- 참여기관: COLOR_SECONDARY 박스
- 지원/감리: COLOR_NEUTRAL 점선 박스

**의사결정 및 협력 프로세스:**
- 정기 회의 주기
- 이슈 에스컬레이션 경로

**슬라이드**: 추진체계 조직도 (1장) + 역할 분담표 (1장)

### 2-6. 세부과제 연구개발계획 (6항)

각 세부과제에 대해 1–2장 슬라이드:
- 세부과제명 및 담당기관
- 세부 연구 목표 (정량)
- 연구 내용 (3–5개 항목)
- 연차별 수행 내용
- 세부과제별 예산 (개략)

**슬라이드**: 세부과제 개요표 (1장) + 세부과제별 상세 (과제 수 × 1–2장)

---

## Step 3 — PPTX 슬라이드 생성

**슬라이드 순서:**
```
[섹션 표지] "연구 계획"
  → 1. 연구기간 및 소요예산 (1–2장)
  → 2. 추진 일정 계획 (2–3장: 간트차트 + 마일스톤)
  → 3. 성과평가 계획 (2장)
  → 4. 기술적 접근방법 및 수행방안 (3–5장)
  → 5. 연구개발업무 추진체계 (2장)
  → 6. 세부과제 연구개발계획 ([세부과제수 × 1–2장])
```

---

## Step 4 — 저장 및 로그

1. 사용자 검토 승인 (Rule 1)
2. `[작업폴더]/[과제명약어]_연구계획.pptx` 저장
3. `YYMMDD_HHMM_research_plan.md` 분석 저장 (Rule 2)
4. `0.FilesUpdate.xlsx` 업데이트 (Rule 4)

---

## 작성 품질 기준

- 일정: WBS와 연동하여 누락 작업 없도록
- 예산: 합계가 맞는지 반드시 검산
- KPI: 과제 개요의 목표 수치와 **일치**
- 추진체계: 역할 중복·공백 없이 MECE하게 분배
- 세부과제: 과제 간 인터페이스(연계 사항) 명시
