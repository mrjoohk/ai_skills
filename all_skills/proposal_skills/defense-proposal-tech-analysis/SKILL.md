# Defense Proposal — 기술 현황 분석 스킬

방산 연구 제안서의 **기술 현황 분석** 섹션(7개 항목)을 작성한다.
이 섹션은 "우리가 어떤 기술을 가지고 있고, 무엇을 개발해야 하며, 어떻게 개발할 것인가"를 체계적으로 보여준다.

## 섹션 구성

| 항 | 제목 | 역할 |
|---|---|---|
| 1 | WBS | 작업 분류 체계 — 연구 범위를 계층적으로 분해 |
| 2 | 구성품별 개발 방법 | 각 구성품의 개발 접근 방식 |
| 3 | TBS | 기술 분류 체계 — 소요 기술을 계층적으로 분해 |
| 4 | 소요 기술 설명 및 현황 | 필요한 모든 기술의 상세 설명 |
| 5 | 소요 기술 분석 | 기술별 특성 및 개발 방향 |
| 6 | 기확보 기술 분석 | 이미 보유한 기술 상세 분석 |
| 7 | 미확보 기술 분석 | 개발이 필요한 기술 상세 분석 |

---

## Step 1 — 체계 분해 정보 수집

사용자로부터 또는 과제 개요에서 수집:
- 개발 대상 체계의 주요 구성품 목록
- 각 구성품의 기능/역할
- 보유 중인 기술 목록 (개략적으로도 가능)

---

## Step 2 — WBS 및 TBS 구조 설계

### 2-1. WBS (Work Breakdown Structure)

**계층 구조 원칙:**
```
Level 0: 최종 산출물 (연구과제명)
Level 1: 주요 구성 체계 (3–6개)
Level 2: 서브 시스템 / 구성품 (각 L1 아래 2–4개)
Level 3: 단위 작업 / 태스크 (선택적)
```

**WBS 표 형식:**
| WBS 코드 | 항목명 | 수준 | 담당 | 비고 |
|---|---|---|---|---|
| 1.0 | [체계명] | L0 | — | — |
| 1.1 | [서브시스템 1] | L1 | 주관 | — |
| 1.1.1 | [구성품 1-1] | L2 | 주관 | — |

**시각화**: defense-wbs-tbs 스킬로 계층 트리 다이어그램 생성.
또는 defense-diagram 스킬로 (C) 시스템 구성도 유형 활용.

### 2-2. TBS (Technology Breakdown Structure)

**계층 구조 원칙:**
```
Level 0: 핵심 기술 영역
Level 1: 기술 그룹
Level 2: 단위 기술
```

**각 단위 기술에 대해 기록:**
- TBS 코드
- 기술명
- 기술 설명 (1–2문장)
- 현재 TRL (1–9)
- 목표 TRL
- 확보/미확보 여부

**TBS 표 형식:**
| TBS 코드 | 기술명 | 현재 TRL | 목표 TRL | 확보 여부 |
|---|---|---|---|---|
| T1.1 | [기술명] | 4 | 7 | 미확보 |

---

## Step 3 — 구성품별 개발 방법 (2항)

각 구성품에 대해:
- **개발 방식**: 자체 개발 / 구매 / 기술도입 / 공동개발
- **기술 접근**: 핵심 알고리즘 또는 설계 방법
- **검증 방법**: 시뮬레이션 / 랩 시험 / 환경 시험 등

**표 형식:**
| 구성품 | 개발 방식 | 핵심 기술 | 검증 방법 | 비고 |
|---|---|---|---|---|

---

## Step 4 — 소요 기술 분석 (4–5항)

### 소요 기술 설명 (4항)
- TBS의 각 단위 기술을 상세 설명
- 기술별 1개 슬라이드 또는 표로 정리
- 내용: 기술 정의 / 원리 / 적용 방식 / 성능 기준

### 소요 기술 분석 (5항)
- 기술별 기술적 난이도 및 위험도
- 개발 방향 및 핵심 성공 요인
- **defense-diagram 스킬** 로 기술 분류 다이어그램 생성

**기술 위험도 매트릭스:**
| 기술명 | 기술 난이도 | 일정 위험 | 기술 위험도 | 대응 방안 |
|---|---|---|---|---|
| [기술명] | 高/中/低 | 高/中/低 | 高/中/低 | [방안] |

---

## Step 5 — 기확보 기술 분석 (6항)

**기확보 기술**: 보유 또는 조달 가능한 기술

각 기술에 대해:
- 기술명 및 TBS 코드
- 확보 경로 (자체 개발 완료 / 기존 연구과제 / 구매 가능)
- 현재 TRL
- 활용 계획 (이번 연구에서 어떻게 사용할지)
- 필요한 개량 사항 (있으면)

**슬라이드**: 기확보 기술 목록 표 + 핵심 기술 설명 (1–2장)

---

## Step 6 — 미확보 기술 분석 (7항)

**미확보 기술**: 신규 개발이 필요한 기술

각 기술에 대해:
- 기술명 및 TBS 코드
- 미확보 이유 (국내 부재 / 기술 수준 미달 / 신개념)
- 개발 방안 (자체 개발 / 기술도입 / 산·학·연 협력)
- 개발 일정 및 마일스톤
- 핵심 기술 도전 사항

**표 형식:**
| 기술명 | 현재 TRL | 목표 TRL | 개발 방안 | 소요 기간 |
|---|---|---|---|---|

**기확보 vs 미확보 비교 다이어그램:**
defense-diagram 스킬로 (E) 기술 비교도 유형 사용.
확보 기술: COLOR_SUCCESS(#2E7D51), 미확보: COLOR_WARNING(#B85C1A) 적용.

---

## Step 7 — PPTX 슬라이드 생성

**슬라이드 순서:**
```
[섹션 표지] "기술 현황 분석"
  → 1. WBS (1–2장: 계층도 + 표)
  → 2. 구성품별 개발 방법 (1–2장)
  → 3. TBS (1–2장: 계층도 + 표)
  → 4. 소요 기술 설명 및 현황 (2–4장)
  → 5. 소요 기술 분석 (1–2장: 분류도 + 위험도 표)
  → 6. 기확보 기술 분석 (1–2장)
  → 7. 미확보 기술 분석 (1–2장)
```

**디자인 특이 사항:**
- 기확보: 녹색 계열 테두리 또는 배지 사용 (#2E7D51)
- 미확보: 주황/빨간 계열 테두리 또는 배지 사용 (#B85C1A)
- TRL 표시: 숫자 배지 형식 (색상으로 단계 구분)

## 작성 품질 기준

- WBS/TBS 코드: 반드시 체계적이고 일관된 번호 체계 사용
- TRL 수준: 1–9 기준 엄격히 적용 (애매한 경우 낮게 평가)
- 미확보 기술: 개발 방안이 구체적이어야 함 (막연한 "검토" 금지)
- 기확보 vs 미확보: 경계가 명확히 구분되어야 함
